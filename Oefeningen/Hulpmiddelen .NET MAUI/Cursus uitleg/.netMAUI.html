Wat is een Framework?

Een framework is een verzameling tools, bibliotheken en regels waarmee je de structuur bepaalt van een applicatie of een deel van een applicatie. Een applicatie met het .NET MAUI framework is anders opgebouwd dan een applicatie met het MVC framework.

Framework vs. Library

Een framework is niet hetzelfde als een library (bibliotheek). Een library is ook herbruikbare code, maar bepaalt niet de structuur van je applicatie. Bij een framework volg je de regels en structuur van het framework. Een library voegt specifieke functionaliteit toe.

Vergelijk het met een huis: het framework is het bouwplan van de architect. Een library is als een exclusieve designfauteuil die je toevoegt.

Voorbeelden van Frameworks

  • Web: Django (Python), Laravel (PHP), Blazor (.NET)
  • JavaScript: Angular, Vue.js
  • Mobile: Flutter, React Native, .NET MAUI
  • Backend: Express.js (Node.js), Spring (Java)

Voorbeelden van Libraries

  • JavaScript: JQuery, Bootstrap
  • Python: Numpy, Matplotlib
  • .NET: Newtonsoft.Json, NUnit

Wat is .NET MAUI?

.NET MAUI staat voor ".NET Multi-platform App User Interface". Het is een open-source framework voor het ontwikkelen van mobiele en desktopapplicaties die kunnen draaien op meerdere platformen zoals Android, Windows, iOS en meer. Met behulp van .NET en C# bouw je één enkele codebase die kan worden gedeeld tussen de verschillende platformen. Op die manier bespaar je ontwikkeltijd en onderhoud.

Waarom .NET MAUI?

Het doel van .NET MAUI is om multi-platform ontwikkeling eenvoudiger te maken. In plaats van dat je vier verschillende UI-frameworks en programmeertalen moet leren, biedt .NET MAUI één uniform framework om zowel mobiele als desktopapps te maken.

Vereisten en Installatie

Om met .NET MAUI te beginnen, heb je Visual Studio 2022 nodig met de juiste componenten.

  1. Installeer Visual Studio 2022: Download en installeer Visual Studio 2022.
  2. Selecteer de juiste Workload: Start de "Visual Studio Installer". Zorg ervoor dat de workload ".NET Multi-platform App UI development" is aangevinkt. Dit installeert automatisch de .NET 8.0 SDK en alle andere benodigdheden.
  3. Verifieer de installatie: Na installatie kun je in Visual Studio een nieuw project aanmaken en controleren of de ".NET MAUI App" template beschikbaar is.

Een Project Aanmaken

Maak in Visual Studio een nieuw project aan van het type ".NET MAUI App" en geef het de naam 'MauiIntroductie'.

Belangrijk!

Gebruik nooit een underscore (_) in een projectnaam van .NET MAUI. Dit kan problemen veroorzaken bij het deployen van de applicatie.

Inhoud van het Projectbestand (.csproj)

Wanneer je dubbelklikt op je project in Visual Studio, open je het `.csproj` bestand. Dit bevat de configuratie van je project.

<PropertyGroup>
    <!-- ... -->
    <TargetFrameworks>net8.0-android;net8.0-ios;net8.0-maccatalyst</TargetFrameworks>
    <TargetFrameworks Condition="$([MSBuild]::IsOSPlatform('windows'))">$(TargetFrameworks);net8.0-windows10.0.19041.0</TargetFrameworks>
    
    <ApplicationTitle>MauiIntroductie</ApplicationTitle>
    <ApplicationId>com.companyname.mauiintroductie</ApplicationId>
    <ApplicationDisplayVersion>1.0</ApplicationDisplayVersion>
    <ApplicationVersion>1</ApplicationVersion>
    <!-- ... -->
</PropertyGroup>
  • TargetFrameworks: De platformen waarvoor je app wordt gebouwd (Windows, Android, iOS, etc.).
  • ApplicationTitle: De naam van je applicatie.
  • ApplicationId: Een unieke identifier voor je applicatie.
  • ApplicationDisplayVersion / ApplicationVersion: Versienummers.

Projectstructuur en Configuratie

Mappenstructuur

  • Platforms: Bevat platformspecifieke code en configuratie.
  • Resources: Bevat gedeelde assets zoals fonts, afbeeldingen (die automatisch worden geresized) en stijlen.

.NET MAUI Startup (MauiProgram.cs)

Dit bestand configureert en start je applicatie. Hier registreer je bijvoorbeeld fonts, services en logging.

public static MauiApp CreateMauiApp()
{
    var builder = MauiApp.CreateBuilder();
    builder
        .UseMauiApp<App>()
        .ConfigureFonts(fonts =>
        {
            fonts.AddFont("OpenSans-Regular.ttf", "OpenSansRegular");
            fonts.AddFont("OpenSans-Semibold.ttf", "OpenSansSemibold");
        });

#if DEBUG
    builder.Logging.AddDebug();
#endif

    return builder.Build();
}

Shell (AppShell.xaml)

De Shell definieert de hoofdstructuur en navigatie van de app. De extensie `.xaml` staat voor eXtensible Application Markup Language, een opmaaktaal om de UI te ontwerpen.

<Shell
    x:Class="MauiApp.AppShell"
    xmlns="http://schemas.microsoft.com/dotnet/2021/maui"
    xmlns:x="http://schemas.microsoft.com/winfx/2009/xaml"
    xmlns:local="clr-namespace:MauiIntroductie"
    Shell.FlyoutBehavior="Disabled">

    <ShellContent
        Title="Home"
        ContentTemplate="{DataTemplate local:MainPage}"
        Route="MainPage" />
</Shell>

Pagina's (.xaml en .xaml.cs)

Een pagina bestaat uit een `.xaml`-bestand voor de layout en een `.xaml.cs`-bestand (code-behind) voor de logica. De code-behind is een `partial class`, wat betekent dat de klassedefinitie is opgesplitst over meerdere bestanden.

Pagina's Toevoegen

Pagina's (ook wel 'Views' genoemd) zijn de schermen van je applicatie. Het is een goede gewoonte om deze in een `Views`-map te plaatsen.

Om een nieuwe pagina toe te voegen, klik je met de rechtermuisknop op de `Views`-map en kies je `Add > New Item...`. Selecteer dan een `.NET MAUI ContentPage (XAML)`.

Naamgevingsconventies

  • Model: Enkelvoud, PascalCase (bv. `Klant.cs`)
  • View: Eindigt op `Page` (bv. `KlantIngevenPage.xaml`)
  • ViewModel: Eindigt op `ViewModel` (bv. `KlantViewModel.cs`)

Navigatie Tussen Pagina's

Nadat je de standaard `MainPage` hebt verwijderd en een nieuwe `HomePage` hebt toegevoegd, moet je de `AppShell.xaml` aanpassen om de nieuwe pagina als startpunt te gebruiken en navigatie mogelijk te maken.

  1. Voeg een XML-namespace toe voor je `Views`-map.
  2. Stel `Shell.FlyoutBehavior` in op `Flyout` om een hamburgermenu te creëren.
  3. Pas de `ShellContent` aan om naar je nieuwe `HomePage` te verwijzen.
<?xml version="1.0" encoding="UTF-8" ?>
<Shell
    x:Class="MauiIntroductie.AppShell"
    xmlns="http://schemas.microsoft.com/dotnet/2021/maui"
    xmlns:x="http://schemas.microsoft.com/winfx/2009/xaml"
    xmlns:views="clr-namespace:MauiIntroductie.Views"
    Shell.FlyoutBehavior="Flyout"
    Title="MauiIntroductie">
    
    <ShellContent
        Title="Home"
        ContentTemplate="{DataTemplate views:HomePage}"
        Route="HomePage" />
    
    <!-- Voeg hier nieuwe pagina's toe voor het menu -->
    
</Shell>

Controls

Een Control is een visueel element op een pagina waarmee de gebruiker kan interacteren. Je bouwt je UI op door controls in XAML te definiëren. Enkele basiscontrols zijn:

  • Label: Toont tekst.
  • Entry: Een invoerveld voor tekst.
  • Button: Een klikbare knop.
  • Image: Toont een afbeelding.
  • Slider: Een schuifbalk.
  • Picker: Een dropdown-keuzevak.

Layouts

Een `ContentPage` kan maar één kind-element bevatten. Om meerdere controls te plaatsen, gebruik je een layout. Een layout kan op zijn beurt meerdere controls (inclusief andere layouts) bevatten.

Verticale StackLayout

Plaatst controls onder elkaar.

<StackLayout>
    <Label Text="Item 1" />
    <Label Text="Item 2" />
</StackLayout>

HorizontalStackLayout

Plaatst controls naast elkaar.

<HorizontalStackLayout>
    <Label Text="Links" />
    <Label Text="Rechts" />
</HorizontalStackLayout>

Grid (Raster)

Positioneert controls in rijen en kolommen. Je definieert de `RowDefinitions` en `ColumnDefinitions` en plaatst elke control in een specifieke cel met `Grid.Row` en `Grid.Column`.

<Grid RowDefinitions="*,*" ColumnDefinitions="*,*">
    <Label Text="A" Grid.Row="0" Grid.Column="0" />
    <Label Text="B" Grid.Row="0" Grid.Column="1" />
    <Label Text="C" Grid.Row="1" Grid.Column="0" />
    <Label Text="D" Grid.Row="1" Grid.Column="1" />
</Grid>

MVVM in .NET MAUI

Bij het ontwikkelen van .NET MAUI-applicaties wordt de gebruikersinterface vaak in XAML opgebouwd. Naarmate een app groeit, kan de code-behind aanpak leiden tot onderhoudsproblemen. Het Model-View-ViewModel (MVVM)-patroon biedt een oplossing door de logica van de toepassing strikt te scheiden van de presentatie-laag. Dit maakt de code gemakkelijker te onderhouden, uit te breiden en te testen.

Wat is MVVM?

MVVM is een architectuurpatroon dat een app opsplitst in drie componenten:

  • Model: De data- en businesslogica-laag. Dit zijn niet-visuele klassen die de gegevens van de applicatie bevatten (bv. een `Product` class). Het model weet niets van de ViewModel of de View.
  • View: De presentatie-laag, verantwoordelijk voor de UI die de gebruiker ziet (meestal in XAML). De View bevat idealiter geen bedrijfslogica en is gekoppeld aan een ViewModel via databinding.
  • ViewModel: De tussenschakel die de staat en interactielogica van de UI bevat. Het exposeert properties en commands waarop de View kan binden, en haalt data op uit het Model (of repositories). Het ViewModel heeft geen directe kennis van UI-elementen.

Deze scheiding maakt de app beter onderhoudbaar, testbaar en maakt parallelle ontwikkeling door designers en ontwikkelaars makkelijker.

Data binding in .NET MAUI

Databinding is het mechanisme dat de View en ViewModel met elkaar verbindt. Elk UI-element heeft een `BindingContext` die je instelt op een ViewModel-instantie. UI-elementen kunnen dan binden aan de properties van dat ViewModel met de `{Binding ...}` syntax.

<!-- De BindingContext wordt ingesteld op het MainViewModel -->
<ContentPage.BindingContext>
    <vm:MainViewModel />
</ContentPage.BindingContext>

<!-- De Text-eigenschap van de Entry is nu gebonden aan de UserName-property in het ViewModel -->
<Entry Text="{Binding UserName}" Placeholder="Typ je naam" />
<Label Text="{Binding UserName, StringFormat='Hallo {0}!'}" />

Bindingen kunnen `OneWay` (van ViewModel naar View) of `TwoWay` (beide richtingen) zijn. Voor invoervelden zoals `Entry` is `TwoWay` de standaard.

ViewModel-eigenschappen en INotifyPropertyChanged

Om de UI automatisch te laten updaten, implementeren ViewModels de `INotifyPropertyChanged` interface. Wanneer een property-waarde verandert, wordt een `PropertyChanged` event getriggerd, wat de UI vertelt om zichzelf bij te werken.

De .NET Community Toolkit MVVM vereenvoudigt dit enorm. Door je ViewModel te laten erven van `ObservableObject` en properties te markeren met `[ObservableProperty]`, wordt alle benodigde code automatisch gegenereerd.

public partial class MyViewModel : ObservableObject
{
    [ObservableProperty]
    private string name; // Genereert een 'Name' property voor databinding
}

Commands en acties (ICommand)

In plaats van event handlers in de code-behind, gebruiken we in MVVM Commands. Een command is een actie in het ViewModel die wordt aangeroepen door de View. Een `Button` heeft bijvoorbeeld een `Command`-eigenschap die je kunt binden aan een `ICommand` in je ViewModel.

De Community Toolkit biedt het `[RelayCommand]` attribuut om eenvoudig commands te genereren vanuit methodes.

public partial class MyViewModel : ObservableObject
{
    [RelayCommand]
    private void SaveData()
    {
        // Logica om data op te slaan...
    }
}
<!-- Dit roept de SaveData-methode in het ViewModel aan -->
<Button Text="Opslaan" Command="{Binding SaveDataCommand}" />

De toolkit genereert automatisch een `SaveDataCommand` property.

ObservableCollection en lijsten

Om een lijst van data te tonen die automatisch wordt bijgewerkt in de UI, gebruik je een `ObservableCollection`. Wanneer je items toevoegt aan of verwijdert uit deze collectie in het ViewModel, wordt de gebonden `CollectionView` in de UI direct bijgewerkt.

public partial class MyViewModel : ObservableObject
{
    public ObservableCollection<string> Tasks { get; } = new();

    [RelayCommand]
    private void AddTask(string task)
    {
        Tasks.Add(task);
    }
}

Dependency Injection en Services

.NET MAUI heeft ingebouwde ondersteuning voor Dependency Injection (DI). Dit patroon stelt je in staat om afhankelijkheden (zoals data services of repositories) aan je ViewModels te "injecteren" via de constructor. Dit ontkoppelt je ViewModel van concrete implementaties en maakt je code beter testbaar.

Je registreert je services en ViewModels in `MauiProgram.cs`:

// In MauiProgram.cs
builder.Services.AddSingleton<INoteRepository, FileNoteRepository>();
builder.Services.AddTransient<NotesViewModel>();

En injecteert ze in de constructor van je ViewModel:

public class NotesViewModel : ObservableObject
{
    private readonly INoteRepository _repo;
    public NotesViewModel(INoteRepository repo)
    {
        _repo = repo;
    }
}

Voorbeeld: MVVM in de Praktijk

Laten we een eenvoudige takenlijst-app bouwen.

TodoViewModel.cs

public partial class TodoViewModel : ObservableObject
{
    [ObservableProperty]
    private string newTaskText;

    public ObservableCollection<string> Tasks { get; } = new();

    [RelayCommand]
    private void AddTask()
    {
        if (!string.IsNullOrWhiteSpace(NewTaskText))
        {
            Tasks.Add(NewTaskText);
            NewTaskText = string.Empty; // Maakt het invoerveld leeg via databinding
        }
    }
}

TodoPage.xaml

<ContentPage ... x:DataType="viewmodels:TodoViewModel">
    <ContentPage.BindingContext>
        <viewmodels:TodoViewModel />
    </ContentPage.BindingContext>

    <VerticalStackLayout Padding="20" Spacing="10">
        <Entry Text="{Binding NewTaskText}" Placeholder="Nieuwe taak..." />
        <Button Text="Toevoegen" Command="{Binding AddTaskCommand}" />
        <CollectionView ItemsSource="{Binding Tasks}">
            <CollectionView.ItemTemplate>
                <DataTemplate>
                    <Label Text="{Binding .}" Padding="5" FontSize="18" />
                </DataTemplate>
            </CollectionView.ItemTemplate>
        </CollectionView>
    </VerticalStackLayout>
</ContentPage>

Dit voorbeeld combineert een `ObservableProperty` voor het invoerveld, een `RelayCommand` voor de knop en een `ObservableCollection` voor de lijst, wat resulteert in een volledig functionele MVVM-implementatie zonder code-behind.

CollectionViews & Models

Het laatste deel van MVVM zijn Models. Dit zijn de klassen die de structuur van je data bepalen. Om een lijst van models te tonen, gebruiken we een `CollectionView`.

BaseViewModel

Het is een goede gewoonte om een `BaseViewModel` aan te maken. Andere ViewModels erven hiervan, zodat je herbruikbare code (zoals een titel-property of een 'Ga Terug'-commando) op één plek kunt centraliseren.

public partial class BaseViewModel: ObservableObject
{
    [ObservableProperty]
    string title;

    [RelayCommand]
    public static async Task GoBackAsync()
    {
        await Shell.Current.GoToAsync("..");
    }
}

ObservableCollections

Om een lijst te binden aan een `CollectionView`, gebruik je een `ObservableCollection`. Dit is een speciale collectie die de UI automatisch op de hoogte stelt wanneer items worden toegevoegd of verwijderd, zonder dat je de hele lijst opnieuw hoeft te laden.

De CollectionView Control

De `CollectionView` heeft enkele belangrijke eigenschappen:

  • ItemsSource: Bind je aan de `ObservableCollection` in je ViewModel.
  • ItemTemplate: Een sjabloon (`DataTemplate`) dat bepaalt hoe elk afzonderlijk item in de lijst eruitziet.
  • SelectedItem: Het item dat de gebruiker selecteert. Dit kun je binden aan een property in je ViewModel.
  • Header / Footer: Statische content boven of onder de lijst.
  • EmptyView: Tekst of een layout die wordt getoond als de lijst leeg is.

Repositories en Interfaces

De interactie met een externe databron (zoals een database) wordt verzorgd door een Repository. Dit is een laag tussen het ViewModel en de data. Door een Interface te gebruiken, definiëren we wat de repository moet doen, zonder te specificeren hoe. Dit ontkoppelt het ViewModel van de concrete data-implementatie, wat de code flexibeler en beter testbaar maakt.

Interfaces

Een interface is een contract. We maken een `Data` map aan met daarin `IRepositories` en `Repositories` submappen. Eerst definiëren we de interface.

IWerknemerRepository.cs

namespace MauiIntroductie.Data.IRepositories
{
    public interface IWerknemerRepository
    {
        public List<Werknemer> GetWerknemers();
    }
}

Repositories

De repository implementeert de interface en bevat de logica om de data daadwerkelijk op te halen (in dit voorbeeld, een hard-coded lijst).

WerknemerRepository.cs

namespace MauiIntroductie.Data.Repositories
{
    public class WerknemerRepository: IWerknemerRepository
    {
        public List<Werknemer> Werknemers = new()
        {
            new Werknemer() { Id = 1, Voornaam = "John", Achternaam = "Doe", ... },
            // ... meer werknemers ...
        };

        public List<Werknemer> GetWerknemers()
        {
            return Werknemers;
        }
    }
}

Dependency Injection

In `MauiProgram.cs` registreren we de interface met haar concrete implementatie. Dit zorgt voor een losse koppeling.

builder.Services.AddSingleton<IWerknemerRepository, WerknemerRepository>();

Vervolgens injecteren we de interface in de constructor van het ViewModel. Het ViewModel weet nu alleen van de interface, niet van de concrete `WerknemerRepository`.

WerknemerViewModel.cs (Constructor)

private readonly IWerknemerRepository _werknemerRepository;

public WerknemerViewModel(IWerknemerRepository werknemerRepository)
{
    Title = "Werknemers";
    _werknemerRepository = werknemerRepository;
    Werknemers = new ObservableCollection<Werknemer>(_werknemerRepository.GetWerknemers());
    Werknemer = new();
}

Praktijkvoorbeelden: MVVM

Hier passen we alle concepten toe in de `WerknemerPage`.

Picker binden aan een Repository

In plaats van de items voor een `Picker` hard in XAML te coderen, halen we ze uit een `FunctieRepository`.

  1. Maak een `Functie` model, `IFunctieRepository` en `FunctieRepository`.
  2. Registreer de service in `MauiProgram.cs`.
  3. Injecteer `IFunctieRepository` in `WerknemerViewModel` en vul een `ObservableCollection`.
  4. Pas het `Werknemer` model aan zodat `Functie` van het type `Functie` is.
  5. Bind de `Picker` in XAML aan de nieuwe collectie.

WerknemerPage.xaml (Picker)

<Picker Title="Selecteer een functie"
        ItemsSource="{Binding Functies}" 
        ItemDisplayBinding="{Binding Naam}" 
        SelectedItem="{Binding Werknemer.Functie}" />

`ItemDisplayBinding` vertelt de `Picker` welke property van het `Functie` object getoond moet worden.

Het "Equals" Probleem

Als je een werknemer uit de lijst selecteert, zal de `Picker` niet correct worden ingesteld. Dit komt omdat .NET niet weet hoe het twee `Functie`-objecten moet vergelijken. De oplossing is om de `Equals`-methode in het `Functie`-model te overschrijven.

// In Functie.cs
public override bool Equals(object? obj)
{
    return obj is Functie functie && functie.Naam == Naam;
}

Nu werkt de `Picker` correct bij het selecteren van een bestaande werknemer. [Afbeelding van een .NET MAUI-app die werknemers uit een repository laadt]

Events en Code-behind Logica

Hoewel MVVM met `Commands` de voorkeur heeft, kun je nog steeds traditionele events gebruiken in de code-behind voor simpele UI-logica of wanneer MVVM overkill is.

XAML:

<Button Text="Klik mij!" Clicked="OnButtonClicked" />
<Label x:Name="resultLabel" Text="Resultaat verschijnt hier" />

C# Code-behind:

private void OnButtonClicked(object sender, EventArgs e)
{
    resultLabel.Text = "De knop is aangeklikt!";
}

Styling en Thema's

Om een consistente look & feel te garanderen, kun je `Styles` gebruiken. Een stijl is een verzameling van property-waarden die je kunt hergebruiken. Stijlen worden meestal gedefinieerd in een `ResourceDictionary`.

Impliciete Stijl (geldt voor alle Buttons):

<ContentPage.Resources>
    <Style TargetType="Button">
        <Setter Property="CornerRadius" Value="10" />
        <Setter Property="BackgroundColor" Value="#3498db" />
        <Setter Property="TextColor" Value="White" />
    </Style>
</ContentPage.Resources>

Expliciete Stijl (handmatig toepassen):

<ContentPage.Resources>
    <Style x:Key="TitleLabelStyle" TargetType="Label">
        <Setter Property="FontSize" Value="24" />
        <Setter Property="FontAttributes" Value="Bold" />
    </Style>
</ContentPage.Resources>

<!-- Toepassen van de stijl -->
<Label Text="Welkom" Style="{StaticResource TitleLabelStyle}" />

Oefeningen

Test je kennis met de volgende opdrachten:

1. Repository voor Producten

Maak een `Product`-model (`Naam`, `Prijs`), een `IProductRepository` en een `ProductRepository` met een hard-coded lijst producten. Maak een `ProductViewModel` dat de producten ophaalt via de repository en weergeeft in een `CollectionView` op een nieuwe `ProductPage`.

2. Item Verwijderen

Voeg een `VerwijderProductCommand` toe aan je `ProductViewModel`. Dit commando moet een product als parameter accepteren. Voeg in de `ItemTemplate` van je `CollectionView` een knop toe die dit commando aanroept, en geef het huidige item mee met `{Binding .}`.

<Button Text="Verwijder" Command="{Binding Source={RelativeSource AncestorType={x:Type viewmodel:ProductViewModel}}, Path=VerwijderProductCommand}" CommandParameter="{Binding .}" />
3. Geselecteerd Item Tonen

Zorg ervoor dat wanneer je een product selecteert in de `CollectionView`, de details (naam en prijs) in aparte `Label`-controls elders op de pagina worden getoond. Gebruik hiervoor de `SelectedItem` binding van de `CollectionView`.
