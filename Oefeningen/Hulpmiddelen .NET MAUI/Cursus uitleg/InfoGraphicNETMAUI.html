De Kern van .NET MAUI Architectuur

Een visueel overzicht van de patronen en datastromen voor het bouwen van robuuste, onderhoudbare cross-platform applicaties.

1. Het MVVM Patroon: De Blauwdruk

Het Model-View-ViewModel (MVVM) patroon is de fundering. Het scheidt de applicatie in drie kerncomponenten, wat leidt tot schonere code, betere testbaarheid en eenvoudiger onderhoud.

View

(XAML)

De "domme" UI-laag. Bevat alleen wat de gebruiker ziet en hoe.

Data Binding

Commands

ViewModel

(C#)

De "lijm". Beheert de UI-staat en -logica. Weet niets van de View.

Vraagt Data

Krijgt Data

Model

(C#)

De "data". Pure datastructuren en business logic.

De ViewModel implementeert INotifyPropertyChanged om de View te vertellen dat data is gewijzigd, en gebruikt ICommand om acties (zoals knopklikken) af te handelen.

2. De Datastroom: Binding & Collections

Data Binding Modes

Data Binding synchroniseert de View en ViewModel. De Mode bepaalt de richting van de datastroom. Dit diagram toont een typische verdeling van het gebruik.

Binding Context

De BindingContext is de sleutel. Het vertelt een View welke ViewModel zijn databron is. Dit wordt meestal in de code-behind van de View ingesteld.

public UserDetailsPage(UserDetailsViewModel vm) { InitializeComponent(); BindingContext = vm; }

Dynamische Lijsten

Gebruik ObservableCollection<T> in plaats van List<T> in je ViewModel. Het geeft automatisch een signaal aan de CollectionView wanneer items worden toegevoegd of verwijderd, waardoor de UI magisch update.

3. De Service Laag: Structuur & Services

Goede architectuur isoleert verantwoordelijkheden. Het Repository Patroon scheidt datalogica van de UI, en Dependency Injection (DI) koppelt alles netjes aan elkaar.

Repository Patroon

De ViewModel kent alleen de Interface (het 'contract'), niet de Repository (de 'implementatie'). Dit maakt de app testbaar en flexibel.

Dependency Injection (DI)

DI is de 'monteur' die de ViewModel automatisch de UserRepository geeft. Je registreert services in MauiProgram.cs met een specifieke levensduur.

4. De Navigatie Stroom: Pagina's & Data

.NET MAUI Shell beheert de navigatie. Vanuit een ViewModel navigeer je met Shell.Current.GoToAsync() en kun je data op twee manieren meesturen.

Pad 1: Simpele Data (Query Property)

Voor simpele waarden zoals een ID. Stuur het mee in de URL-string.

1. Navigeren (ViewModel)

await Shell.Current.GoToAsync( $"{nameof(DetailsPage)}?UserId=123" );
↓

2. Ontvangen (DetailsViewModel)

[QueryProperty(nameof(UserId), "UserId")] public partial class DetailsViewModel ... { [ObservableProperty] string userId; // Wordt "123" }

Pad 2: Complexe Data (Object)

Voor volledige objecten. Stuur een Dictionary mee.

1. Navigeren (ViewModel)

var user = new User { Name = "Jan" }; await Shell.Current.GoToAsync( nameof(DetailsPage), new Dictionary<string, object> { { "SelectedUser", user } } );
↓

2. Ontvangen (DetailsViewModel)

[QueryProperty(nameof(SelectedUser), "SelectedUser")] public partial class DetailsViewModel ... { [ObservableProperty] User selectedUser; // Wordt het 'user' object }
